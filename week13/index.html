ESP32 Multi-Channel Controller

Interactive Generative Art with Analog & Digital Inputs

Processing & Arduino Serial Communication Project

🎥 Final Results

This project successfully demonstrates the control of three distinct visual parameters in Processing using physical hardware: Color, Position, and Object Scale.

Demo 1: Joystick controlling Color (X-Axis) and Position (Y-Axis).
Demo 2: Ultrasonic Sensor controlling Object Scale (Distance).

🛠️ Project Overview

The Goal

The objective was to build a multi-channel controller that allows a user to change various parameters in a Processing sketch. A key requirement was to experiment with sensors other than potentiometers.

Hardware Components

  • Microcontroller: ESP32 (Adafruit HUZZAH32)
  • Sensor 1 (Analog): XY Joystick Module (Controls Position and Color).
  • Sensor 2 (Digital): HC-SR04 Ultrasonic Distance Sensor (Controls Size).

⚠️ The Process & Troubleshooting

Integrating the digital ultrasonic sensor with the analog joystick presented significant technical challenges. Below is the breakdown of how I solved them.

The Problem: Unstable Sensor Readings
Initially, the HC-SR04 ultrasonic sensor was returning `0` values consistently. After researching the datasheet, I realized the sensor requires 5V to operate, but the ESP32 GPIO pins only provide 3.3V.

The Solution:

I re-routed the sensor's VCC connection. Instead of using the breadboard's 3V rail, I connected the sensor directly to the ESP32's USB pin. This bypassed the voltage regulator and provided the necessary 5V from the USB connection, resulting in stable distance readings.

Secondary Issue (Signal Interference):

I also discovered that Pin 13 is connected to the on-board LED, which interfered with the sensor's Trigger signal. I re-mapped the Trigger to Pin 14 to ensure clean data transmission.

💻 Code Logic

Arduino Side: The code reads three values in a loop: Distance (cm), X-Axis (0-4095), and Y-Axis (0-4095). It formats these into a single string (e.g., "15,2048,1023") and sends it via Serial.

Processing Side: The sketch reads the string, splits the data by commas, and maps the values to visual parameters:

  • Distance → Maps to Circle Diameter (50px - 300px).
  • Joystick X → Maps to RGB Color values.
  • Joystick Y → Maps to Vertical Position (Y-coordinate).
